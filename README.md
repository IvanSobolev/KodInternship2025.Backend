# KodInternship2025.Backend

## Описание проекта

Этот проект представляет собой веб-приложение для управления персоналом и задачами, разработанное с использованием технологии ASP.NET Core 8.0. Система предоставляет RESTful API для взаимодействия с базой данных и обеспечивает обработку событий в реальном времени через WebSockets и Kafka.

### Структура проекта

Проект разделен на следующие основные модули:

- **Demo.API**: API-уровень, содержащий контроллеры, обработчики запросов и интеграцию с Kafka
  - Controllers: REST API контроллеры
  - Hubs: SignalR хабы для реализации реального времени
  - Kafka: Интеграция с Apache Kafka
  - Managers: Бизнес-логика приложения

- **Demo.DAL**: Уровень доступа к данным
  - Abstractions: Интерфейсы и абстракции
  - Dto: Объекты передачи данных
  - Enums: Перечисления
  - Models: Модели данных
  - Repositories: Репозитории для работы с данными
  - Migrations: Миграции базы данных

### Технологический стек

- **Backend**: ASP.NET Core 8.0, C# 12.0
- **База данных**: PostgreSQL
- **Обмен сообщениями**: Apache Kafka
- **Реальное время**: SignalR
- **Контейнеризация**: Docker, Docker Compose
- **CI/CD**: Настроено для автоматического развертывания

## Почему такая архитектура?

Выбранная архитектура имеет множество преимуществ, которые делают её идеальным решением для современных веб-приложений:

1. **Микросервисная архитектура** обеспечивает:
   - Масштабируемость: каждый компонент может масштабироваться независимо
   - Устойчивость: отказ одного сервиса не приводит к отказу всей системы
   - Гибкость в разработке: разные команды могут работать над разными частями системы

2. **Использование Kafka** дает следующие преимущества:
   - Асинхронное взаимодействие между сервисами
   - Надежность доставки сообщений и устойчивость к сбоям
   - Возможность сохранения и воспроизведения истории событий
   - Высокая пропускная способность для обработки больших объемов данных

3. **Трехуровневая архитектура** (API, Managers, DAL):
   - Четкое разделение ответственности
   - Повышение тестируемости кода
   - Упрощение поддержки и расширения системы
   - Возможность замены компонентов без изменения всей системы

4. **Контейнеризация с Docker**:
   - Одинаковое окружение для разработки и продакшн
   - Простое масштабирование
   - Быстрый запуск и остановка сервисов
   - Изоляция зависимостей

5. **Использование SignalR для реального времени**:
   - Мгновенное обновление UI без перезагрузки страницы
   - Двусторонняя связь между клиентом и сервером
   - Автоматическое переключение между WebSockets, Server Sent Events, Long Polling

Данная архитектура обеспечивает не только техническую эффективность, но и бизнес-преимущества, позволяя быстро реагировать на изменения требований и быстро внедрять новые функции.
